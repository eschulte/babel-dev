TODO can't use cl functions

  1. Packages that are part of Emacs are not allowed to load cl at

runtime. So you can use (eval-when-compile (require 'cl)) in order to get the cl macros expanded, but you cannot use cl functions. Annoying yes - but this is how it is. Basically, any cl functions we want to use we have to find alternatives for it in normal emacs Lisp code, or write our own org-… version.

If the compiler is nice enough to complain about these at compile time, I guess we'll just wait and let it sort this out for us.

TODO unique ob-*.el file names

  1. All file names in Emacs must be 8+3 unique, so the first 8

characters of a file name must make it unique. File names org-babel- something.el will not work.
